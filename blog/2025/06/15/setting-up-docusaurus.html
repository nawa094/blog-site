When I began this project, I thought: "This will be easy - I'll just use DeepSeek R1 to generate the configs and be done in minutes." The initial setup looked promising:

npx create-docusaurus@latest my-blog classic --typescript

But reality had other plans...

The Configuration Hurdlesâ

1. GitHub Pages Deployment Woesâ

The first version of my deploy.yml failed with frustrating permissions errors:

# First attempt (failed)
- name: Deploy to GitHub Pages
  uses: peaceiris/actions-gh-pages@v3
  with:
    github_token: ${{ secrets.GITHUB_TOKEN }}

After consulting the Docusaurus deployment docs, I fixed it by adding:

# Working version
permissions:
  contents: write # Critical for gh-pages branch
  pages: write

2. The Mysterious 404â

Even after successful builds, my site showed only the repository README. The solution? Two crucial changes:

  1. Repository naming: Had to rename to <username>.github.io
  2. Config updates:
// docusaurus.config.js
baseUrl: '/', // Not '/my-blog/'
url: 'https://nawa094.github.io'

The Debugging Processâ

Here's what ultimately worked:

  1. Local testing first:
npm run build && npm run serve

  1. Manual inspection of the gh-pages branch contents

  2. Cache busting by adding ?v=2 to the URL during testing

Final Config Snippetsâ

Working deploy.ymlâ

name: Deploy to GitHub Pages
on:
  push:
    branches: [main]
permissions:
  contents: write
  pages: write
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
      - run: npm install
      - run: npm run build
      - uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./build
